## 一、课程背景——数学是计算的底层语言

现代设备能够识别人脸、预测天气、自动驾驶、处理海量数据——这一切背后的基础是数学。从推荐你下一首可能喜欢的歌，到优化最快的配送路线，数学是计算的核心引擎。

本课程属于"计算数学（Mathematics for Computing）"系列的线性代数部分，聚焦三个核心数据结构：

| 结构                     | 维度                    | 类比         | 应用                           |
| ------------------------ | ----------------------- | ------------ | ------------------------------ |
| **向量（Vector）** | 一维——一个数字列表    | 一行数据     | 搜索匹配、特征表示             |
| **矩阵（Matrix）** | 二维——数字的网格/表格 | 一张电子表格 | 数据变换、传感器融合、图像处理 |
| **张量（Tensor）** | 多维——矩阵的堆叠      | 一个魔方     | 推荐系统、深度学习             |

它们是所有机器学习和数据处理的基础构件。

## 二、向量（Vector）——搜索与匹配

### 2.1 场景引入：电商搜索

课程用一个叫 Nile 的电商应用作为贯穿案例。作为机器学习工程师，我们要解决三个核心任务：

1. **搜索匹配**：用户输入关键词，找到最相关的商品 → 用向量
2. **相关商品推荐**：找到与当前商品相似的其他商品 → 用矩阵
3. **个性化推荐**：根据用户历史行为预测喜好 → 用张量

### 2.2 为什么需要向量——计算机只懂数字

人类理解"蓝色健身包"这个概念，但计算机不懂文字——它只认识数字。要让计算机理解搜索词，我们必须把文字属性翻译成数字，这就是 **特征编码（Feature Encoding）**。

课程中的编码规则：

**包类型编码：**

| 类型               | 编码 |
| ------------------ | ---- |
| 双肩包（Backpack） | 01   |
| 健身包（Gym Bag）  | 02   |
| 手提包（Tote）     | 03   |

**颜色编码（三位数字，留出色调变化空间）：**

| 颜色               | 编码 |
| ------------------ | ---- |
| 红色（Red）        | 150  |
| 绿色（Green）      | 250  |
| 浅蓝（Light Blue） | 415  |
| 蓝色（Blue）       | 450  |
| 深蓝（Dark Blue）  | 490  |

**价格：** 直接使用数值，去掉货币符号。$23.40 → 23.4

### 2.3 构建向量

有了编码规则，就可以把每个商品和每个搜索词表示为一个 **向量**——一组有序的数字列表。

**商品向量示例：**

```
蓝色健身包 $23.40  →  [02, 450, 23.4]
                       类型  颜色  价格

粉色双肩包 $34.00  →  [01, 350, 34.0]
红色手提包 $42.60  →  [03, 150, 42.6]
浅蓝健身包 $23.00  →  [02, 415, 23.0]
```

**查询向量（Query Vector）：**

当用户搜索 "blue gym bag" 时，系统创建查询向量：`[02, 450, _]`

- `02` = 健身包
- `450` = 蓝色
- `_`（下划线）= 对价格没有要求，任何值都可以

### 2.4 搜索过程——逐步精化

Alice 在 Nile 上搜索的完整过程：

```
搜索词: "bag"
查询向量: [_, _, _]        → 所有类型、所有颜色、所有价格
结果: 返回所有包（双肩包、健身包、手提包...）→ 太泛了

搜索词: "gym bag"
查询向量: [02, _, _]       → 类型=健身包，其余不限
结果: 只返回健身包 → 好多了

搜索词: "blue gym bag"
查询向量: [02, 450, _]     → 类型=健身包，颜色=蓝色
结果: 只返回蓝色健身包 → 精准匹配
```

每次用户添加更多信息，查询向量就更加具体，搜索结果也更加精准,多维度描述,缩小查询范围

### 2.5 向量比较——如何找到"最接近"的商品

系统将查询向量与所有商品向量进行比较，找到"距离最近"的商品。课程提到了 **点积（Dot Product）** 作为比较方式，这里做详细补充。

#### 点积（Dot Product）

两个向量的点积是对应元素相乘后求和：

```
向量 A = [a₁, a₂, a₃]
向量 B = [b₁, b₂, b₃]

A · B = a₁×b₁ + a₂×b₂ + a₃×b₃
```

**示例：** 查询向量 `[02, 450]`（忽略价格）与两个商品比较：

```
蓝色健身包: [02, 450]
A · B = 02×02 + 450×450 = 4 + 202500 = 202504  ← 高分

红色手提包: [03, 150]
A · B = 02×03 + 450×150 = 6 + 67500 = 67506    ← 低分
```

点积越大，两个向量越"相似"。

#### 余弦相似度（Cosine Similarity）——更常用的比较方式（补充）

在实际搜索系统中，**余弦相似度** 比原始点积更常用，因为它不受向量长度（数值大小）的影响，只关注"方向"是否一致：

```
cos(θ) = (A · B) / (|A| × |B|)

其中 |A| = √(a₁² + a₂² + a₃²)  是向量 A 的长度（模）
```

结果在 -1 到 1 之间：

- 1 = 完全相同方向（最相似）
- 0 = 正交（无关）
- -1 = 完全相反

### 2.6 向量的数学定义（补充）

课程为了应用场景做了简化，这里补充向量的正式定义：

**向量** 是一个有 **大小（Magnitude）** 和 **方向（Direction）** 的数学对象。在计算中，我们通常把它简化为一个有序的数字列表（数组）。

```
标量（Scalar）:  一个单独的数字        → 42
向量（Vector）:  一组有序的数字        → [02, 450, 23.4]
                 可以理解为 n 维空间中的一个点或一个方向
```

**向量的维度（Dimension）** = 向量中数字的个数。`[02, 450, 23.4]` 是一个 3 维向量。

常见向量运算：

| 运算               | 公式                                       | 示例                     |
| ------------------ | ------------------------------------------ | ------------------------ |
| **加法**     | [a₁,a₂] + [b₁,b₂] = [a₁+b₁, a₂+b₂] | [1,2] + [3,4] = [4,6]    |
| **标量乘法** | k × [a₁,a₂] = [k×a₁, k×a₂]          | 2 × [3,4] = [6,8]       |
| **点积**     | a₁×b₁ + a₂×b₂ + ...                  | [1,2]·[3,4] = 3+8 = 11  |
| **模/长度**  | √(a₁² + a₂² + ...)                    | \|[3,4]\| = √(9+16) = 5 |

## 三、矩阵（Matrix）——多维比较与数据变换

### 3.1 从向量到矩阵——为什么需要升维

向量很棒，但它一次只能比较一个特征。Alice 找到了一个浅蓝色健身包 $23，现在她想找"同样颜色、同样类型、价格相近"的其他包——需要同时比较多个特征。

这就是矩阵的用武之地：**矩阵是二维的数字网格，可以同时组织和比较多个对象的多个特征。**

```
             颜色    价格
商品 A  →  [ 415,   23.0 ]     ← 每一行是一个商品（一个向量）
商品 B  →  [ 420,   25.0 ]     ← 每一列是一个特征
商品 C  →  [ 450,   22.5 ]
商品 D  →  [ 415,   24.0 ]

这整个就是一个 4×2 的矩阵（4行2列）
```

矩阵让系统一次性比较所有商品在所有特征上的差异，而不需要逐个特征单独过滤——更快、更智能。

### 3.2 矩阵的数学定义

**矩阵** 是一个按行和列排列的数字矩形阵列。

```
    列1  列2  列3
行1 [ 5    2    3 ]
行2 [ 1    4    6 ]

这是一个 2×3 矩阵（2行3列），记作 m×n
```

**方阵（Square Matrix）**：行数 = 列数的矩阵。如 2×2、3×3。

### 3.3 应用场景：自动驾驶传感器融合

课程的第二个大案例是自动驾驶汽车。自动驾驶车配备多种传感器：

| 传感器                      | 功能                               |
| --------------------------- | ---------------------------------- |
| **摄像头（Camera）**  | 像人眼一样捕获视觉图像             |
| **雷达（Radar）**     | 检测周围物体                       |
| **激光雷达（LiDAR）** | 发射激光脉冲测量距离，生成 3D 点云 |

问题是：每个传感器安装在车的不同位置，看到的"视角"不同。左传感器看到行人在右边，右传感器看到行人在左边——但行人其实就在正前方。

**解决方案：用矩阵运算将所有传感器的数据统一到同一个坐标系中。**

#### 传感器数据矩阵

一个 LiDAR 传感器检测到三个物体，数据存储为矩阵：

```
              距离(m)  高度(m)  是否移动(1=是,0=否)
树木      →  [  5       3        0  ]
红绿灯    →  [  8       4        0  ]
行人      →  [  3       1.7      1  ]

这是一个 3×3 的数据矩阵
```

#### 变换矩阵（Transformation Matrix）

每个传感器有自己的 **变换矩阵**，用于校正该传感器的位置和视角偏差：

```
变换矩阵 T = [ 1.1    0    0  ]
             [  0    1/3   0  ]
             [  0     0    1  ]
```

各元素的含义：

- `1.1`（第一列）：将距离值增加 10%，校正传感器的测距偏差
- `1/3`（第二列）：将高度值缩小到 0-1 之间（传感器最高能看到 3 米，除以 3 做归一化）
- `1`（第三列）：运动指标不变

**每个传感器都有自己的变换矩阵。** 所有传感器的数据分别乘以各自的变换矩阵后，就被统一到同一个坐标系中，可以合并成一张完整的环境地图。

### 3.4 矩阵乘法——完整规则

#### 方阵乘法（Square Matrices）

两个 2×2 矩阵相乘：

```
A = [ 5  2 ]      B = [ 3  4 ]
    [ 1  4 ]          [ 2  6 ]

A × B 的计算方法：
第一个矩阵的每一行 × 第二个矩阵的每一列
对应位置相乘后求和

结果矩阵 C:

C[1,1] = 5×3 + 2×2 = 15 + 4  = 19
C[1,2] = 5×4 + 2×6 = 20 + 12 = 32
C[2,1] = 1×3 + 4×2 = 3 + 8   = 11
C[2,2] = 1×4 + 4×6 = 4 + 24  = 28

C = [ 19  32 ]
    [ 11  28 ]
```

**步骤总结：** 结果矩阵第 i 行第 j 列的值 = 第一个矩阵第 i 行与第二个矩阵第 j 列的点积。

3×3 矩阵同理，只是组合更多（9 个元素各算一次）。

#### 矩阵乘法的核心规则

**矩阵乘法可行的条件：第一个矩阵的列数必须等于第二个矩阵的行数。**

```
  A          B          结果
(m × n)  ×  (n × p)  =  (m × p)
         ↑       ↑
      这两个数必须相等！
```

| A 的大小 | B 的大小 | 能否相乘 | 结果大小     |
| -------- | -------- | -------- | ------------ |
| 2×2     | 2×2     | ✅       | 2×2         |
| 2×3     | 3×2     | ✅       | 2×2         |
| 2×2     | 3×2     | ❌       | —           |
| 3×3     | 3×1     | ✅       | 3×1         |
| 1×4     | 4×1     | ✅       | 1×1（标量） |

**矩阵乘法不满足交换律！** `A × B ≠ B × A`（大多数情况下）。

#### 自动驾驶中的矩阵乘法应用

将传感器数据矩阵乘以变换矩阵：

```
数据矩阵 D (3×3)  ×  变换矩阵 T (3×3)  =  校正后的数据 (3×3)

[ 5    3    0 ]     [ 1.1   0    0 ]     [ 5.5   1     0 ]
[ 8    4    0 ]  ×  [  0   1/3   0 ]  =  [ 8.8   1.33  0 ]
[ 3    1.7  1 ]     [  0    0    1 ]     [ 3.3   0.57  1 ]

树木: 距离校正为5.5m, 高度归一化为1
红绿灯: 距离8.8m, 高度1.33
行人: 距离3.3m, 高度0.57, 正在移动
```

每个传感器的数据都经过各自的变换矩阵处理后，结果可以合并成一张统一的环境地图。

### 3.5 矩阵求逆——修正运动带来的失真

#### 问题：车辆移动导致数据失真

自动驾驶车在行驶中持续采集传感器数据。但车辆移动会导致数据失真——车向前开了 2 米后，物体距离变近了，传感器看到的物体"变大了"（就像你把手从远处移到眼前，手看起来变大了一样）。

传感器不知道车移动了，它只记录看到的东西。所以移动后的数据需要修正。

```
移动前的传感器读数:
[ 5    3  ]    树木: 距离5m, 高度3m
[ 3    1.7]    行人: 距离3m, 高度1.7m

移动后的传感器读数（失真了）:
[ 3    3.6]    树木: 距离变近(3m), 但高度看起来变高了(3.6m)?!
[ 1    2.1]    行人: 距离变近(1m), 高度看起来也变高了(2.1m)?!
```

高度不应该变——树和人没有长高。失真来自于距离变化影响了高度的感知。

#### 运动矩阵（Movement Matrix）

车辆的运动效应被记录为一个矩阵：

```
运动矩阵 M = [ 1    0  ]
             [ 0.2  1  ]

0.2 表示: 部分距离数据"泄漏"到了高度维度，
          导致物体看起来比实际更高
```

#### 逆矩阵（Inverse Matrix）——"撤销"运动的影响

要修正失真，需要找到运动矩阵的 **逆矩阵（Inverse Matrix）**——它能"撤销"运动矩阵的效果。

**核心类比：**

- 数字 5 的逆是 1/5，因为 5 × 1/5 = 1
- 矩阵 M 的逆是 M⁻¹，因为 M × M⁻¹ = I（单位矩阵）

#### 单位矩阵（Identity Matrix）

单位矩阵是矩阵世界的"1"——对角线上全是 1，其余全是 0：

```
2×2 单位矩阵:    3×3 单位矩阵:
I = [ 1  0 ]     I = [ 1  0  0 ]
    [ 0  1 ]         [ 0  1  0 ]
                      [ 0  0  1 ]
```

**关键性质：任何矩阵乘以单位矩阵等于它本身。** A × I = A，就像 a × 1 = a。

#### 2×2 矩阵求逆公式

对于矩阵 A = `[a b; c d]`：

```
A⁻¹ = (1 / det(A)) × adj(A)

其中:
  det(A) = ad - bc        ← 行列式（Determinant）
  adj(A) = [ d   -b ]     ← 伴随矩阵（Adjoint）
           [ -c   a ]        交换 a 和 d，b 和 c 变符号
```

**步骤：**

1. 计算行列式 det(A) = ad - bc
2. 如果 det(A) = 0 → **矩阵不可逆！**（无法求逆）
3. 构造伴随矩阵：交换主对角线元素，副对角线元素取反
4. 乘以 1/det(A)

#### 完整求逆示例

```
矩阵 A = [ 2  3 ]
         [ 1  4 ]

步骤1: det(A) = 2×4 - 3×1 = 8 - 3 = 5  ✅ 不为零，可以求逆

步骤2: adj(A) = [ 4   -3 ]
               [ -1    2 ]

步骤3: A⁻¹ = (1/5) × [ 4   -3 ] = [ 4/5    -3/5 ]
                      [ -1    2 ]   [ -1/5    2/5 ]

验证: A × A⁻¹ 应该等于单位矩阵 I

[ 2  3 ] × [ 4/5   -3/5 ] = [ 2×4/5+3×(-1/5)    2×(-3/5)+3×2/5 ]
[ 1  4 ]   [ -1/5   2/5 ]   [ 1×4/5+4×(-1/5)    1×(-3/5)+4×2/5 ]

                             = [ 8/5-3/5    -6/5+6/5 ]
                               [ 4/5-4/5    -3/5+8/5 ]

                             = [ 5/5    0   ] = [ 1  0 ]  ✅ 单位矩阵！
                               [  0    5/5 ]   [ 0  1 ]
```

#### 行列式为零 → 不可逆

```
矩阵 B = [ 2  4 ]
         [ 3  6 ]

det(B) = 2×6 - 4×3 = 12 - 12 = 0

无法计算 1/0 → 这个矩阵没有逆矩阵！
```

**如何判断一个矩阵是否可逆？** 计算行列式，如果不为零就可逆。

课程中的练习：判断哪些矩阵可逆：

```
[ 2  4 ]  det = 2×6 - 4×3 = 0   ❌ 不可逆
[ 3  6 ]

[ 1  2 ]  det = 1×4 - 2×3 = -2  ✅ 可逆
[ 3  4 ]

[ 3  6 ]  det = 3×4 - 6×2 = 0   ❌ 不可逆
[ 2  4 ]
```

#### 应用逆矩阵修正传感器数据

```
运动矩阵 M = [ 1    0  ]
             [ 0.2  1  ]

步骤1: det(M) = 1×1 - 0×0.2 = 1  ✅

步骤2: M⁻¹ = (1/1) × [  1     0 ] = [  1     0 ]
                      [ -0.2   1 ]   [ -0.2   1 ]

步骤3: 失真数据 × M⁻¹ = 修正后数据

[ 3    3.6 ] × [  1     0  ] = [ 3×1+3.6×(-0.2)     3×0+3.6×1   ]
[ 1    2.1 ]   [ -0.2   1  ]   [ 1×1+2.1×(-0.2)     1×0+2.1×1   ]

                               = [ 3-0.72     3.6 ] = [ 2.28   3.6 ]
                                 [ 1-0.42     2.1 ]   [ 0.58   2.1 ]
```

修正后，高度值更接近移动前的原始值。在实际系统中，数字精度更高、矩阵更大，但原理完全相同。

### 3.6 矩阵运算规则总结（补充）

| 运算                | 规则                                                               |
| ------------------- | ------------------------------------------------------------------ |
| **加法/减法** | 只有同型矩阵（行列数相同）才能相加减，对应位置元素运算             |
| **标量乘法**  | 每个元素乘以同一个标量                                             |
| **矩阵乘法**  | A(m×n) × B(n×p) = C(m×p)，第一个矩阵的列数必须等于第二个的行数 |
| **交换律**    | ❌ 矩阵乘法不满足交换律：A×B ≠ B×A                              |
| **结合律**    | ✅ (A×B)×C = A×(B×C)                                           |
| **分配律**    | ✅ A×(B+C) = A×B + A×C                                          |
| **转置**      | 行变列、列变行：(A^T)ᵢⱼ = Aⱼᵢ                                  |
| **逆矩阵**    | 只有方阵且行列式≠0才有逆；A × A⁻¹ = I                          |

## 四、张量（Tensor）——多维数据与推荐系统

### 4.1 为什么需要张量

向量是一维的（一列数字），矩阵是二维的（行×列）。但有些数据天然具有更多维度。

回到 Nile 电商的例子：我们不仅要比较商品的特征（矩阵能做到），还要考虑 **用户的行为历史**——不同用户对不同商品有不同的交互方式（点击、搜索、加入购物车、评分、评价）。

这就需要在矩阵的基础上再加一个维度——得到 **张量（Tensor）**。

### 4.2 什么是张量

**张量是矩阵的多维推广。**

```
标量（Scalar）:  0 维  →  一个数字           42
向量（Vector）:  1 维  →  一列数字           [1, 2, 3]
矩阵（Matrix）:  2 维  →  行 × 列            [[1,2],[3,4]]
张量（Tensor）:  3+维  →  多个矩阵堆叠起来    想象一个"魔方"
```

课程的类比：**张量就像一个魔方（Rubik's Cube）**——每一"层"是一个矩阵，代表一种不同的数据维度。

### 4.3 推荐系统中的张量

在 Nile 的推荐系统中，张量的三个维度分别是：

```
维度1 (行): 用户
维度2 (列): 商品
维度3 (层): 用户行为类型

         商品A  商品B  商品C
        ┌──────────────────┐
用户1   │  3     0     1   │  ← 第1层: 点击次数
用户2   │  0     5     2   │
        └──────────────────┘
        ┌──────────────────┐
用户1   │  1     0     0   │  ← 第2层: 搜索次数
用户2   │  0     3     1   │
        └──────────────────┘
        ┌──────────────────┐
用户1   │  1     0     0   │  ← 第3层: 加入购物车
用户2   │  0     1     1   │
        └──────────────────┘
        ... 还可以有更多层: 评分、评价、购买...
```

### 4.4 Alice 的个性化推荐

系统分析 Alice 的张量数据后，可以给出智能推荐：

| 推荐商品   | 推荐理由                                   |
| ---------- | ------------------------------------------ |
| 蓝色双肩包 | Alice 之前买过双肩包（购买历史）           |
| 蓝色运动鞋 | Alice 搜索和点击过很多蓝色商品（颜色偏好） |
| 健身器材   | 与当前浏览的健身包相关（品类关联）         |

**系统不是在"猜测"——它在从张量中的行为模式中"学习"。** 张量让推荐系统能够同时考虑"谁"、"对什么商品"、"做了什么行为"这三个维度的交叉关系。

### 4.5 张量在更广泛领域的应用（补充）

| 领域                   | 张量的角色                                                                           |
| ---------------------- | ------------------------------------------------------------------------------------ |
| **图像处理**     | 一张彩色图片就是一个 3D 张量：高度 × 宽度 × 颜色通道（RGB=3层）                    |
| **视频处理**     | 4D 张量：高度 × 宽度 × 颜色通道 × 时间帧                                          |
| **自然语言处理** | 一批文本数据：批大小 × 句子长度 × 词向量维度                                       |
| **深度学习**     | 神经网络的权重、输入和输出都用张量表示。PyTorch 和 TensorFlow 的核心数据结构就是张量 |

**TensorFlow 这个名字本身就来源于"张量在计算图中流动"。**

## 五、从数据到智能系统——全景总结

### 5.1 三种数据结构的递进关系

```
问题复杂度递增 ──────────────────────────────►

向量 (Vector)         矩阵 (Matrix)         张量 (Tensor)
一维数字列表          二维数字网格            多维数字结构

[02, 450, 23.4]      [ 5    3    0 ]       多个矩阵堆叠
                      [ 8    4    0 ]
                      [ 3   1.7   1 ]

搜索匹配              数据变换                推荐系统
特征表示              传感器融合              深度学习
词向量                图像像素矩阵           视频数据
```

### 5.2 自动驾驶中的矩阵应用总结

```
多个传感器各自采集数据
        │
        ▼
每个传感器的数据 × 各自的变换矩阵 = 统一坐标系的数据
        │
        ▼
合并所有传感器数据 → 完整的环境地图
        │
        ▼
车辆移动 → 数据失真
        │
        ▼
失真数据 × 运动矩阵的逆矩阵 = 修正后的数据
        │
        ▼
准确的环境感知 → 安全的驾驶决策
```

- **矩阵乘法**：合并和变换传感器数据
- **矩阵求逆**：修正运动带来的数据失真

没有矩阵运算，自动驾驶汽车就必须逐个处理数百万个数据点，实时决策将不可能实现。

### 5.4 核心要点

1. **计算机只认识数字**——文字、图像、声音等所有数据都必须被编码为数字（向量/矩阵/张量）才能被处理
2. **向量是最基本的数据表示**——一组有序数字，用于表示商品特征、搜索词、词义等
3. **矩阵是二维的向量集合**——可以同时比较多个对象的多个特征，也可以用来变换数据（旋转、缩放、校正）
4. **矩阵乘法的前提条件**：第一个矩阵的列数 = 第二个矩阵的行数
5. **矩阵求逆可以"撤销"一个变换**——但前提是行列式不为零
6. **张量是矩阵的高维推广**——处理同时涉及多个维度（用户、商品、行为）的复杂数据
7. **这三种结构是机器学习的基础语言**——搜索引擎、推荐系统、自动驾驶、图像识别、自然语言处理全都建立在它们之上

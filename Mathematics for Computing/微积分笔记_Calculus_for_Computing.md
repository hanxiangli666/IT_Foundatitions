## 一、导数（Derivative）——度量变化的工具

### 1.1 为什么需要导数

自动驾驶汽车没有眼睛，也没有直觉，但它必须做到：
- 保持安全距离
- 平滑调整速度
- 及时停车

要实现这些，它必须 **追踪速度每一秒钟的变化**——而这正是导数的工作。

**导数衡量的是一个量变化的快慢。** 就像"速度"衡量"距离随时间变化的快慢"一样，导数是"变化率"的数学表达。

### 1.2 从直线到曲线——Julia 的故事

#### 匀速运动 = 直线 = 常数导数

Julia 开车去图书馆，一路堵车，以恒定速度行驶：每 15 秒走 30 米。

```
速度 = 距离变化 / 时间变化 = 30m / 15s = 2 m/s
```

在距离-时间图上，匀速运动画出一条 **直线**。直线的 **斜率（Slope）** 就是速度，也就是导数。

```
距离(m)
  │      ╱  斜率 = 2 m/s = 导数
90├────╱
  │  ╱
30├╱
  └──┬──┬──┬── 时间(s)
     15 30 45
```

#### 停下 = 平线 = 导数为零

45 秒后 Julia 停了下来，停了 15 秒。图上这段是 **水平线**——距离没有变化，斜率 = 0，导数 = 0。

#### 返程 = 下降线 = 负导数

Julia 掉头慢慢开回去。图上的线 **向下倾斜**——这不是说她在倒车，而是她与起点的距离在减小。向下的斜率意味着 **负导数**——距离在减少。

```
距离(m)
  │      ╱‾‾‾‾╲
90├────╱        ╲
  │  ╱     停    ╲  返程（负斜率）
30├╱              ╲
  └──┬──┬──┬──┬──┬── 时间(s)
     15 30 45 60 75
```

**总结三种状态：**

| 图形特征 | 斜率/导数 | 含义 |
|---------|----------|------|
| 陡峭上升 | 正的、数值大 | 速度快 |
| 水平 | 零 | 停止 |
| 下降 | 负的 | 距离在减小（靠近出发点） |

#### 现实中的曲线

现实中 Julia 不会瞬间从行驶变为停止——她会 **逐渐减速**，速度平滑变化，画出的是 **曲线** 而不是折线。

要找到曲线上某一点的"瞬时速度"，我们画 **切线（Tangent）**——一条刚好在那一点"触碰"曲线的直线。切线的斜率就是该点的导数。

```
距离
  │     ╱‾‾‾╲
  │   ╱  ╱    ╲     ← 切线：该点的斜率 = 该点的瞬时速度
  │  ╱ ╱       ╲
  │╱╱            ╲
  └─────────────── 时间
```

- 切线越陡 → 速度越快
- 切线越平 → 速度越慢
- 切线水平 → 速度为零（停止）
- 切线向下 → 距离在减小

### 1.3 自动驾驶中的导数应用

自动驾驶汽车前方检测到施工区域。距离-时间图呈 U 形：先靠近（距离减小，下坡），经过施工区后远离（距离增大，上坡）。

汽车在 40 米处发现障碍物，系统开始分析：
1. 通过切线斜率判断当前速度
2. 斜率变平 → 正在减速
3. 斜率为零 → 停下来
4. 变道后斜率变陡 → 重新加速

**但计算机不会真的"画切线"——它用代数公式直接计算导数。**

### 1.4 导数的代数计算

#### 符号与定义

| 符号 | 含义 |
|------|------|
| f(x) | 函数——给定 x，计算出一个值 |
| f'(x) 或 df/dx | f(x) 的导数——x 变化时 f(x) 变化的速率 |
| **求导（Differentiation）** | 求导数的过程 |

#### 常数的导数

如果 f(x) = 6（一个常数），无论 x 怎么变，f(x) 都不变。没有变化 → 变化率为零。

```
f(x) = 6  →  f'(x) = 0

f(x) = -3  →  f'(x) = 0

任何常数的导数都是 0
```

#### 直线的导数

f(x) = 3x 是一条直线。斜率（rise/run）恒定为 3。

```
f(x) = 3x  →  f'(x) = 3
```

#### 幂法则（Power Rule）——最核心的求导法则

课程通过观察发现了一个规律：

```
f(x) = x²   →  f'(x) = 2x
f(x) = x³   →  f'(x) = 3x²
f(x) = x⁴   →  f'(x) = 4x³
```

规律：**指数移到前面做系数，原指数减 1**。

**幂法则公式：**

$$f(x) = x^n \quad\Rightarrow\quad f'(x) = n \cdot x^{n-1}$$

**带系数的情况：**

$$f(x) = a \cdot x^n \quad\Rightarrow\quad f'(x) = a \cdot n \cdot x^{n-1}$$

示例：
```
f(x) = 5x³  →  f'(x) = 5 × 3 × x² = 15x²
f(x) = 2x⁵  →  f'(x) = 2 × 5 × x⁴ = 10x⁴
f(x) = 7x   →  f'(x) = 7 × 1 × x⁰ = 7  （x⁰ = 1）
```

**多项式求导——逐项求导：**

```
f(x) = 4x³ + 5x² - 7

逐项求导:
  4x³  →  12x²
  5x²  →  10x
  -7   →  0

f'(x) = 12x² + 10x
```

### 1.5 用导数解决自动驾驶问题

汽车距离施工区域的函数：

```
f(t) = t² - 80t + 400

其中 t 是时间（秒），f(t) 是与施工区域的距离（米）
t=0 时，距离 = 400 米
```

用幂法则求导：

```
f'(t) = 2t - 80
```

这个导数函数就是汽车的"速度函数"——可以直接算出任意时刻的速度：

| 应用 | 计算 | 含义 |
|------|------|------|
| **t=0 时的速度** | f'(0) = 2×0 - 80 = **-80** | 负值 = 距离在减小 = 正在靠近施工区，速度 80 m/s |
| **何时停车？** | 令 f'(t) = 0：2t - 80 = 0 → **t = 40 秒** | 40 秒时速度为零 = 停车 |
| **t < 40 时** | f'(t) < 0 | 正在减速/靠近 |
| **t > 40 时** | f'(t) > 0 | 正在加速/远离 |

导数不只是给出速度的大小——负号告诉你方向（靠近还是远离），比单纯的"速度"信息更丰富。

### 1.6 导数在图像识别中的应用

人类一眼看到"日落"，但计算机看到的是一个数字网格——每个像素有一个亮度值（0=黑色，255=白色）。

```
像素亮度图（一行像素）:
亮度
255│          ╱╲
   │         ╱  ╲         ← 太阳的位置
   │        ╱    ╲
   │ ──────╱      ╲──────
  0└─────────────────────── 像素位置
           亮度突然跳变 = 边缘
```

模型扫描每一行像素，测量亮度变化：
- **亮度突然跳变**（导数值大）→ 说明这里是一条**边缘**（如太阳与天空的分界线）
- **亮度平缓变化**（导数值小）→ 平滑过渡区域
- **亮度不变**（导数为零）→ 均匀区域

甚至一个像素的变化（比如一只飞过的鸟）也能在导数图上产生明显的尖峰。模型对每一行和每一列都做这个分析，就能理解整张图像的结构——检测边缘、识别形状、发现物体。

**补充——图像处理中的经典导数应用：**

| 技术 | 原理 |
|------|------|
| **Sobel 算子** | 用矩阵卷积计算图像在 x 和 y 方向的亮度梯度 |
| **Canny 边缘检测** | 基于梯度大小和方向找到图像中的边缘 |
| **卷积神经网络（CNN）** | 自动学习最有效的"导数滤波器"来识别图像特征 |

## 二、偏导数与梯度（Partial Derivatives & Gradient）

### 2.1 从单变量到多变量

前面的导数只涉及一个变量（如时间 t）。但现实问题通常涉及 **多个变量**。

课程用外卖司机 Suzy 的场景来引入：她要找到最快的送餐路线。路线的"行驶时间"取决于：
- **x**：到目的地的距离（英里）
- **y**：限速

这两个因素同时影响行驶时间，不能只看一个。

### 2.2 偏导数（Partial Derivative）

当函数有多个变量时，**偏导数** 测量的是：在其他变量不变的情况下，改变一个变量会怎样影响结果。

符号：用 ∂（圆体 d）代替普通的 d：

| 符号 | 含义 |
|------|------|
| df/dx | 普通导数——f 只有一个变量 x |
| ∂f/∂x | 偏导数——f 有多个变量，只对 x 求导，其他变量当常数 |
| ∂f/∂y | 偏导数——只对 y 求导，x 当常数 |

#### 计算方法

**核心规则：对哪个变量求导，就把其他变量当常数处理。**

示例 1：f(x,y) = x² + y²

```
∂f/∂x：把 y² 当常数 → 导数为 0
  x² 的导数 = 2x
  → ∂f/∂x = 2x

∂f/∂y：把 x² 当常数 → 导数为 0
  y² 的导数 = 2y
  → ∂f/∂y = 2y
```

示例 2：f(x,y) = x³ + y² + xy

```
∂f/∂x：y 当常数
  x³ → 3x²
  y² → 0（常数的导数为零）
  xy → y（x 的系数是 y，对 x 求导后 y 保留）
  → ∂f/∂x = 3x² + y

∂f/∂y：x 当常数
  x³ → 0
  y² → 2y
  xy → x（y 的系数是 x，对 y 求导后 x 保留）
  → ∂f/∂y = 2y + x
```

**含有混合项（如 xy）时的注意事项（补充）：**

| 项 | 对 x 求偏导（y 当常数） | 对 y 求偏导（x 当常数） |
|---|---|---|
| x³ | 3x² | 0 |
| y² | 0 | 2y |
| xy | y | x |
| 5xy | 5y | 5x |
| x²y | 2xy | x² |

### 2.3 梯度（Gradient）

**梯度 = 所有偏导数组成的向量。**

```
f(x,y) = x² + y²

梯度 ∇f = [∂f/∂x, ∂f/∂y] = [2x, 2y]
```

梯度告诉我们：**在当前位置，函数值增长最快的方向。**

#### 山坡类比

想象你站在山坡上想要尽快到达山脚：
- 你的第一直觉是走最短路径（一个变量：距离）
- 但有些路太陡或太危险，你还需要考虑安全（第二个变量）
- 梯度就是那个 **同时平衡所有因素后的最佳下山方向**

#### 梯度的几何含义

对于 f(x,y) = x² + y²（一个碗形的 3D 曲面）：

```
在点 (1,1)：
  梯度 = [2×1, 2×1] = [2, 2]
  
含义：在这个点，函数值向 x 和 y 的正方向都在增大
      如果想让函数值减小（下山），应该往 [-2, -2] 方向走
```

- 梯度的方向 = 函数增长最快的方向
- 梯度的大小 = 增长的"陡峭程度"
- **负梯度方向 = 函数下降最快的方向**

### 2.4 外卖路线优化实例

Suzy 的路线行驶时间函数：

```
T(x,y) = x⁴ - 3x² + y³ - 5xy + 10

x = 距离（英里），y = 限速等级
```

求梯度：

```
∂T/∂x = 4x³ - 6x - 5y
∂T/∂y = 3y² - 5x

梯度 ∇T = [4x³ - 6x - 5y,  3y² - 5x]
```

在点 (1,1) 代入：

```
∂T/∂x = 4(1)³ - 6(1) - 5(1) = 4 - 6 - 5 = -7
∂T/∂y = 3(1)² - 5(1) = 3 - 5 = -2

梯度 = [-7, -2]
```

两个分量都是负数 → 行驶时间正在沿当前方向减少 → Suzy 应该继续走这条路。

系统每一步都重新计算梯度。如果梯度变为正值（行驶时间开始增加），模型就知道该换路了。

## 三、梯度下降（Gradient Descent）

### 3.1 核心思想

梯度下降是机器学习中最重要的优化算法。思路极其直观：

1. 站在山上某个位置
2. 看看梯度方向（最陡的上坡方向）
3. 往 **相反方向** 迈一步（下坡）
4. 到了新位置，重新算梯度
5. 重复，直到到达谷底（最优解）

```
损失值
  │  ╲
  │   ╲  ← 第1步：梯度大，步幅大
  │    ╲
  │     ╲
  │      ╲ ← 第2步：梯度小些，步幅小些
  │       ╲
  │        ╲_ ← 第3步：快到底了
  │          ╲___
  │              ‾‾‾ ← 到达最低点：梯度≈0，停止
  └────────────────── 参数值
```

### 3.2 数学表达（补充）

参数更新公式：

```
新参数 = 旧参数 - 学习率 × 梯度

θ_new = θ_old - α × ∇f(θ_old)
```

- **θ（theta）**：模型的参数（如神经网络的权重）
- **α（alpha）**：学习率——控制每一步的大小
- **∇f**：损失函数的梯度

#### 学习率（Learning Rate）的重要性

| 学习率 | 后果 |
|--------|------|
| **太大** | 步伐过大，可能跳过最低点，甚至发散 |
| **太小** | 步伐过小，训练极慢，可能陷在局部最低点 |
| **合适** | 稳步下降，收敛到最优解 |

```
学习率太大:              学习率太小:             学习率合适:
  ╲  ╱╲  ╱               ╲                      ╲
   ╲╱  ╲╱                 ╲.                      ╲
    发散了!                  ╲..                    ╲
                               ╲...                  ╲_
                                 太慢了...               ‾‾  ✓
```

### 3.3 梯度下降的实际应用

| 应用 | 怎么用梯度下降 |
|------|--------------|
| **Netflix / Spotify** | 最小化推荐预测误差——预测你会喜欢什么，与实际对比，不断调整 |
| **自动驾驶** | 实时优化速度和转向——最小化与理想轨迹的偏差 |
| **Amazon / Uber Eats** | 最小化配送时间——实时调整路线 |
| **人脸识别** | 最小化识别错误率——调整模型参数直到准确识别 |
| **股票预测** | 最小化预测误差——让模型的预测越来越接近真实走势 |

课程中 Suzy 遇到交通堵塞的场景完美展示了梯度下降的实时特性：
1. 原路线的梯度计算出最优路径
2. 5 分钟后遇到堵车
3. App 重新计算梯度——发现当前方向时间在增加
4. 切换路线，重新开始梯度下降
5. 不是随机换路，而是沿着"时间减少最快"的方向调整

## 四、链式法则与反向传播（Chain Rule & Backpropagation）

### 4.1 复合函数（Composite Function）

**复合函数 = 函数嵌套——一个函数的输出作为另一个函数的输入。**

课程的类比：大盒子里装了一个小盒子。

```
小盒子: y = 3x² + 1       ← 内层函数
大盒子: z = y³ + 3         ← 外层函数

把小盒子放进大盒子（用 y 的定义替换）:
z = (3x² + 1)³ + 3        ← 复合函数
```

在神经网络中，复合函数无处不在：
- 输入乘以权重 → 求和 → 这是一个函数
- 求和结果传入激活函数 → 这是另一个函数
- 一个嵌套另一个

### 4.2 链式法则（Chain Rule）

如果我们想知道"改变 x 会怎样影响最终的 z"，但 x 先影响 y，y 再影响 z，那么：

$$\frac{dz}{dx} = \frac{dz}{dy} \times \frac{dy}{dx}$$

**链式法则：复合函数的导数 = 外层函数的导数 × 内层函数的导数。**

#### 完整示例

```
y = 3x² + 1
z = y³ + 3

求 dz/dx:

步骤1: dy/dx = 6x              （幂法则）
步骤2: dz/dy = 3y²             （幂法则）
步骤3: 把 y 用 x 的表达式替换: dz/dy = 3(3x² + 1)²
步骤4: 链式法则: dz/dx = dz/dy × dy/dx = 3(3x² + 1)² × 6x
                                        = 18x(3x² + 1)²
```

#### 面包店类比

课程用了一个很好的类比：

```
酵母用量(x) ──→ 面团质量(y) ──→ 面包质量(z)
                函数1: y=f(x)    函数2: z=g(y)

面包师想知道: "增加酵母用量会怎样影响面包质量？"

链式法则: dz/dx = dz/dy × dy/dx
= (面包质量对面团质量的变化率) × (面团质量对酵母用量的变化率)
```

通过链式法则计算 dz/dx 并令其等于零，可以找到最佳的酵母用量——再多一点面包质量就开始下降了。

### 4.3 神经网络基础

课程用 Netflix 推荐系统来引入神经网络。

#### 神经网络的三层结构

```
输入层              隐藏层              输出层
(Input)            (Hidden)           (Output)

[类型: 4] ──w₁──→  ┌────────┐
                    │ 激活函数 │──→  [推荐概率: 0.29]
[场景: 2] ──w₂──→  └────────┘

w₁ = 0.2 (类型的权重)
w₂ = 0.6 (场景的权重)
```

| 组件 | 角色 |
|------|------|
| **输入层** | 接收数据（如节目类型=犯罪=4，场景=城市=2） |
| **隐藏层** | 处理数据、发现模式（实际系统有很多层） |
| **输出层** | 给出预测（如观看概率 0.29） |
| **权重（Weight）** | 连接上的数值，表示该输入有多重要 |
| **激活函数** | 决定节点是否"激活"并传递信号 |

#### 前向传播计算

```
步骤1: 加权求和
  S = 输入₁ × 权重₁ + 输入₂ × 权重₂
  S = 4 × 0.2 + 2 × 0.6 = 0.8 + 1.2 = 2

步骤2: 通过激活函数
  y = 3S² + 5S（课程中使用的简化激活函数）
  y = 3(2)² + 5(2) = 12 + 10 = 22
  归一化后 → 0.29 (29%)

步骤3: 与实际值比较
  预测: 0.29（用户不太会看）
  实际: 0.65（用户很喜欢这个节目！）
  
  误差太大！需要调整权重。
```

### 4.4 反向传播（Backpropagation）

#### 核心思想

预测错了怎么办？**把误差"传回去"，调整权重，让下次预测更准。**

```
前向传播: 输入 → 权重 → 激活函数 → 输出（预测值）
                                       ↓
                                   与实际值比较 → 误差
                                       ↓
反向传播: 误差 → 调整权重 → 让输出更接近实际值
```

#### 用链式法则调整权重

要调整权重 w，需要知道"改变 w 会怎样改变最终输出"——这是一个复合函数问题。

把权重 w₁ 替换为变量 w：

```
求和函数: S(w) = 4w + 1.2        ← 内层函数
激活函数: y(S) = 3S² + 5S        ← 外层函数

要求: dy/dw（改变权重 w 对输出 y 的影响）

链式法则: dy/dw = dy/dS × dS/dw
```

分别求导：

```
dS/dw = 4                       （S = 4w + 1.2 对 w 求导）
dy/dS = 6S + 5                  （y = 3S² + 5S 对 S 求导）
```

用 S 的表达式替换：

```
dy/dS = 6(4w + 1.2) + 5 = 24w + 7.2 + 5 = 24w + 12.2
```

链式法则相乘：

```
dy/dw = (24w + 12.2) × 4 = 96w + 48.8
```

代入当前权重 w = 0.2：

```
dy/dw = 96(0.2) + 48.8 = 19.2 + 48.8 = 68
```

但课程中计算得到的是 48（可能使用了略不同的简化），关键理解是：

**导数值为正 → 增大权重会增大输出 → 而我们确实需要增大输出（从 0.29 到 0.65）→ 所以模型增大这个权重。**

模型同时对所有权重做这个计算，用梯度下降同步调整，逐步让预测逼近真实值。

#### 反向传播的完整流程

```
1. 前向传播: 用当前权重计算预测值
2. 计算损失: 预测值与真实值的差距
3. 反向传播: 用链式法则计算每个权重对损失的"贡献"
4. 更新权重: 用梯度下降调整权重，减小损失
5. 重复: 用新权重再做前向传播，再比较，再调整...

经过成千上万次循环，模型越来越准确
```

这就是 Netflix 从"随机推荐"变成"懂你"的过程。

### 4.5 损失函数（Loss Function）（补充）

课程暗示了"预测值与实际值的差距"但没有给出正式名称。这个差距的度量函数叫 **损失函数（Loss Function）**，也叫 **代价函数（Cost Function）**。

常见损失函数：

| 损失函数 | 公式 | 适用场景 |
|---------|------|---------|
| **均方误差 MSE** | (预测值 - 实际值)² 的平均 | 回归问题（预测连续值） |
| **交叉熵 Cross-Entropy** | -[y·log(ŷ) + (1-y)·log(1-ŷ)] | 分类问题（是/否、猫/狗） |

梯度下降的目标就是 **最小化损失函数**——让预测值尽可能接近真实值。

### 核心概念速查

| 概念 | 一句话定义 |
|------|-----------|
| **导数** | 函数在某一点的变化率（斜率） |
| **幂法则** | f(x)=xⁿ → f'(x)=nxⁿ⁻¹，最常用的求导公式 |
| **偏导数** | 多变量函数中，固定其他变量，只对一个变量求导 |
| **梯度** | 所有偏导数组成的向量，指向函数增长最快的方向 |
| **梯度下降** | 沿负梯度方向反复小步移动，找到函数的最小值 |
| **学习率** | 梯度下降中每一步的步幅大小 |
| **复合函数** | 一个函数的输出作为另一个函数的输入（嵌套） |
| **链式法则** | 复合函数的导数 = 外层导数 × 内层导数 |
| **损失函数** | 度量预测值与真实值差距的函数 |
| **反向传播** | 用链式法则计算每个权重对损失的贡献，然后用梯度下降调整权重 |

### 核心要点

1. **导数 = 变化率**：斜率陡=变化快，斜率平=变化慢，斜率零=不变
2. **幂法则让求导变成机械计算**：指数下来做系数，原指数减一
3. **偏导数处理多变量问题**：固定其他变量，一次只看一个的影响
4. **梯度指向"上坡"最快方向**，所以想要"下坡"（最小化损失）就走反方向
5. **梯度下降是机器学习的核心优化算法**——几乎所有模型训练都在用
6. **链式法则解决"嵌套函数"的求导问题**——这在神经网络中无处不在
7. **反向传播 = 链式法则的系统化应用**：把误差从输出层逐层传回，用梯度下降调整每一层的权重
8. **这些数学工具的组合让机器能够"学习"**：自动驾驶、推荐系统、图像识别全都建立在这套框架之上
